# OpenCode TUI for VS Code

> **Disclaimer 1**: This is a personal project built for personal use. It is in active development and may change significantly without notice.

> **Disclaimer 2**: Any PRs or Issues generated by AI agents or automated tools will be closed without review.

Embed [OpenCode](https://opencode.ai) TUI directly in VS Code as a webview panel with proper keyboard handling for **all keyboard layouts** (Cyrillic, Korean, Japanese, etc.).

## Features

- **Full TUI in VS Code** — OpenCode terminal as a sidebar view or tab
- **Layout-agnostic keyboard** — `event.code`-based interception; works with any keyboard layout
- **Leader key support** — Ctrl+X (physical Ctrl position) activates leader mode regardless of layout
- **Attach files** — Right-click files or selections in VS Code to attach them to OpenCode context
- **IDE context awareness** — MCP server exposes current editor state to OpenCode
- **Localized** — UI translated to 10 languages (see [Localization](docs/localization.md))
- **Future-proof** — designed to support future OpenCode versions as long as possible; minimal coupling to internal OpenCode APIs

## Requirements

### Runtime (required to use the extension)

- **VS Code** `^1.84.0`
- **System Node.js** `^18` (for `node-pty` native module — separate process outside VS Code's Electron)
  - Windows: `C:\Program Files\nodejs\node.exe`
  - Linux/macOS: must be available in `PATH`
- **npm packages** (bundled in `.vsix`):
  - `node-pty` `^1.1.0` — PTY emulation for opencode process
  - `@xterm/xterm` `^5.5.0` — terminal renderer
  - `@xterm/addon-fit` `^0.10.0` — auto-fit terminal to container
  - `@modelcontextprotocol/sdk` `^1.29.0` — MCP server for IDE context
- **OpenCode CLI** — must be installed and available:
  - Windows: auto-detected from `%LOCALAPPDATA%\OpenCode\opencode.exe` or configured path
  - Linux/macOS: must be in `PATH`

### Build (required to build from source)

- **Node.js** `^18`
- **npm**
- **Python** + **C++ build tools** (for `node-pty` native compilation on Windows: Visual Studio Build Tools / "Desktop development with C++")

## Installation

### From VSIX

```bash
npm install
npm run build
npx vsce package
```

Install the generated `.vsix` file via VS Code's Extensions view → ... → Install from VSIX.

### From source (development)

```bash
npm install
npm run build
```

Then press `F5` in VS Code to launch the Extension Development Host.

## Configuration

| Setting | Default | Description |
|---------|---------|-------------|
| `opencode-tui-unofficial.opencodePath` | `"opencode"` | Path to the OpenCode CLI executable |
| `opencode-tui-unofficial.serverPort` | `0` | Port for OpenCode server (`0` = random free port) |
| `opencode-tui-unofficial.leaderChords` | `[]` | Leader chord characters to intercept (empty = read from `tui.json` or built-in defaults) |

## Usage

- Open the **OpenCode** panel from the activity bar (secondary sidebar)
- `Ctrl+Shift+P` → **OpenCode: Open Terminal**
- Right-click a file in the explorer → **Attach to OpenCode**
- Select text in an editor → right-click → **Attach to OpenCode**

### Keyboard handling

The extension intercepts keyboard events at the `document` level using `event.code` (physical key position), bypassing the layout-dependent `event.key` that VS Code's built-in terminal relies on. This means:

- **Cyrillic layouts** — Ctrl+ч (physical X) activates leader, not Ctrl+х
- **Korean layouts** — All hotkeys map to physical key positions
- **Any layout** — Leader chord `m` always maps to the physical M key

For full details, see [Keyboard Handling](docs/keyboard-handling.md).

### Project structure

```
├── src/
│   ├── extension.ts              # Entry point, activation
│   ├── types.ts                  # TUI command IDs and shared types
│   ├── opencodeServer.ts         # OpenCode process lifecycle + PTY helper
│   ├── ptyHelper.ts              # node-pty process for terminal emulation
│   ├── httpClient.ts             # Typed REST client for OpenCode API
│   ├── webviewProvider.ts        # Webview panel with xterm.js
│   ├── mcp-server.ts             # MCP server for IDE context
│   ├── vscode-editor-state.ts    # Current editor state provider
│   ├── commands/
│   │   └── attachFile.ts         # Attach file/selection commands
│   └── ui/
│       └── webview.ts            # Frontend: terminal + keyboard interceptor
├── l10n/                         # Localization bundles (10 languages)
├── package.nls.json              # English source for package.json strings
├── package.nls.*.json            # Translations (10 languages)
├── esbuild.mjs                   # Build script (extension + webview + helper)
└── tsconfig.json
```

### Adding custom actions

1. Create a handler in `src/commands/yourAction.ts`
2. Register it in `extension.ts` via the command system
3. Add the keyboard binding in `package.json`

See [Development Guide](docs/development.md) for more details.
