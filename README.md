# OpenCode TUI for VS Code

> **Disclaimer 1**: This is a personal project built for personal use. It is in active development and may change significantly without notice.

> **Disclaimer 2**: Any PRs or Issues generated by AI agents or automated tools will be closed without review.

Embed [OpenCode](https://opencode.ai) TUI directly in VS Code as a webview panel with proper keyboard handling for **all keyboard layouts** (Cyrillic, Korean, Japanese, etc.).

## Features

- **Full TUI in VS Code** ‚Äî OpenCode terminal as a sidebar view or tab, each with its own server instance
- **Toggle focus** ‚Äî `Ctrl+Shift+'` switches between the editor and OpenCode panel (customizable in VS Code keybindings)
- **Leader key support** ‚Äî Ctrl+X (physical Ctrl position) activates leader mode regardless of layout
- **Attach files** ‚Äî Right-click files or selections in VS Code to attach them to OpenCode context
- **IDE context awareness** ‚Äî MCP server exposes current editor state to OpenCode
- **Settings modal** ‚Äî On-screen settings panel in the terminal (gear button in status bar)
- **Localized** ‚Äî UI translated to 10 languages (see [Localization](docs/localization.md))
- **Future-proof** ‚Äî designed to support future OpenCode versions as long as possible; minimal coupling to internal OpenCode APIs

## Requirements

### Runtime (required to use the extension)

- **VS Code** `^1.84.0`
- **System Node.js** `^18` (for `node-pty` native module ‚Äî separate process outside VS Code's Electron)
  - Windows: `C:\Program Files\nodejs\node.exe`
  - Linux/macOS: must be available in `PATH`
- **npm packages** (bundled in `.vsix`):
  - `node-pty` `^1.1.0` ‚Äî PTY emulation for opencode process
  - `@xterm/xterm` `^5.5.0` ‚Äî terminal renderer
  - `@xterm/addon-fit` `^0.10.0` ‚Äî auto-fit terminal to container
  - `@modelcontextprotocol/sdk` `^1.29.0` ‚Äî MCP server for IDE context
- **OpenCode CLI** ‚Äî must be installed and available:
  - Windows: auto-detected from `%LOCALAPPDATA%\OpenCode\opencode.exe` or configured path
  - Linux/macOS: must be in `PATH`

### Build (required to build from source)

- **Node.js** `^18`
- **npm**
- **Python** + **C++ build tools** (for `node-pty` native compilation on Windows: Visual Studio Build Tools / "Desktop development with C++")

## Configuration

| Setting | Default | Description |
|---------|---------|-------------|
| `opencode-tui-unofficial.opencodePath` | `"opencode"` | Path to the OpenCode CLI executable |
| `opencode-tui-unofficial.serverPort` | `0` | Port for OpenCode server (`0` = random free port) |
| `opencode-tui-unofficial.leaderChords` | `[]` | Leader chord characters to intercept (empty = read from `tui.json` or built-in defaults) |
| `opencode-tui-unofficial.ctrlASelectAll` | `true` | Intercept Ctrl+A in the terminal for "select all" instead of sending raw `\x01` |

Settings can also be changed via the in-terminal settings modal (gear button in status bar).

## Usage

- Open the **OpenCode** panel from the activity bar (right sidebar)
- `Ctrl+Shift+P` ‚Üí **OpenCode: Open Terminal**
- Right-click a file in the explorer ‚Üí **Attach to OpenCode**
- Select text in an editor ‚Üí right-click ‚Üí **Attach to OpenCode**
- Click the gear icon (‚öô) in the terminal status bar to open settings

### Keyboard handling

The extension intercepts keyboard events at the `document` level using `event.code` (physical key position), bypassing the layout-dependent `event.key` that VS Code's built-in terminal relies on. This means:

- **Cyrillic layouts** ‚Äî Ctrl+—á (physical X) activates leader, not Ctrl+—Ö
- **Korean layouts** ‚Äî All hotkeys map to physical key positions
- **Any layout** ‚Äî Leader chord `m` always maps to the physical M key

For full details, see [Keyboard Handling](docs/keyboard-handling.md).

### Adding custom actions

1. Create a handler in `src/commands/yourAction.ts`
2. Register it in `extension.ts` via the command system
3. Add the keyboard binding in `package.json`
